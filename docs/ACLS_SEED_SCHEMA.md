# ACLS Scenario Seed — Field Schema (for narrative content)

This is the exact schema for the **updated `acls_scenario_seed.json`** that Cowork
builds from the OCR'd case cards. The importer (`scripts/import-acls-seed.js`,
Path A) now maps these fields into the `scenarios` table, so the case content
shows in our familiar scenario layout (grader Phase 2 + scenario library).

## How it loads
- Run: `node scripts/import-acls-seed.js <seed.json> [--dry-run]` (dry-run first).
- Idempotent: matches each scenario by `(case_code, cert_course)` and updates in
  place — **safe to re-run**. Existing structure + rubric (segments) are untouched;
  this only ADDS the narrative fields.
- Narrative fields go **flat on each scenario object** (alongside `case_code`,
  `name`, `segments`, …) — OR nested under a `content: { … }` object. Either works.
- Any field omitted is simply skipped (no overwrite-to-null). Author what the card
  has; leave the rest out.

## Scenario object — structural keys (KEEP these, already in the seed)
```jsonc
{
  "case_code": "CASE_48",            // natural key — DO NOT change
  "name": "Case 48 — SVT → VF → Asystole → ROSC",
  "cert_course": "acls",
  "cert_tier": "megacode_practice",  // or megacode_testing / learning_station
  "grading_model": "adv_cert_checklist",
  "scenario_scope": "full",
  "segments": [ /* rubric — leave as-is */ ]
  // ↓↓↓ ADD the narrative fields below ↓↓↓
}
```

## Narrative fields — EXACT names, types, formats
Types are enforced by the DB columns; match them or the import row fails.

### Plain text (string)
`chief_complaint`, `dispatch_time`, `dispatch_location`, `dispatch_notes`,
`patient_name`, `patient_sex`, `patient_weight`, `general_impression`,
`environment_notes`, `history`, `patient_presentation`, `instructor_notes`,
`allergies`
```jsonc
"chief_complaint": "Weakness and lightheadedness",
"dispatch_location": "Private residence",
"patient_name": "Richard Pace",
"patient_sex": "male",
"patient_weight": "85 kg",
"allergies": "NKDA",
"instructor_notes": "ACLS MEGACODE — BRADY START\n\nThis megacode crosses ...",
```

### Integer
`patient_age`
```jsonc
"patient_age": 66,
```

### String arrays (JSON array of strings)
`medical_history`, `medications`, `allergies`*, `learning_objectives`,
`critical_actions`, `debrief_points`, `equipment_needed`, `expected_interventions`
```jsonc
"medical_history": ["Prior MI", "CHF", "Hypertension"],
"medications": ["Carvedilol 25mg BID", "Lisinopril 20mg daily"],
"critical_actions": ["Identify 3rd degree block and administer atropine", "..."],
"learning_objectives": ["Manage symptomatic bradycardia with atropine and pacing", "..."],
"debrief_points": ["What rhythm did you start with and how did it change?", "..."],
```
\* `allergies` is a TEXT column — pass a plain string (e.g. `"NKDA"`), not an array.

### JSON objects (jsonb)
`initial_vitals`, `vitals`, `sample_history`, `phases`, `opqrst`,
`secondary_survey`, `ekg_findings`

**`initial_vitals`** — object; fill the keys the card gives, leave others "":
```jsonc
"initial_vitals": {
  "bp": "82/50", "hr": "36", "rr": "", "spo2": "93%", "etco2": "38",
  "temp": "", "loc": "", "pain": "", "skin": "", "pupils": "",
  "ekg_rhythm": "3rd degree AV block", "lung_sounds": "", "blood_glucose": "",
  "gcs_total": "", "other_findings": []
}
```
**`sample_history`** — object:
```jsonc
"sample_history": { "signs_symptoms": "", "events_leading": "", "last_oral_intake": "" }
```
**`phases`** — array of phase objects (the megacode progression). Each phase has a
`name`, optional `onset`, and a `vitals` object (same shape as `initial_vitals`):
```jsonc
"phases": [
  { "name": "Initial Presentation", "onset": "", "vitals": { "bp": "82/50", "hr": "36", "spo2": "93%", "ekg_rhythm": "3rd degree block" } },
  { "name": "Arrest — PEA",          "onset": "after atropine fails", "vitals": { "ekg_rhythm": "PEA" } },
  { "name": "VFib",                   "onset": "", "vitals": { "ekg_rhythm": "VFib" } },
  { "name": "ROSC",                   "onset": "", "vitals": { "bp": "100/60", "hr": "90", "ekg_rhythm": "Sinus" } }
]
```
(The `id` field on phases is auto-generated by the app on edit — omit it in the seed.)

## Minimal viable per case (what to prioritize from the cards)
For Thursday usefulness, the highest-value fields: `chief_complaint`,
`patient_age`/`patient_sex`, `initial_vitals` (esp. `ekg_rhythm`, bp, hr, spo2),
`phases` (the rhythm progression), `critical_actions`, `instructor_notes`. The
rest are bonus.

## Scope
The 12 **megacode** cases (CASE_48–55 practice, MEGACODE_TEST_2/4/9/10 testing)
are the priority. Learning-station cases (CASE_16/17/26/27/34/36/39/40) are
formative and can be narrative-light.

## After Cowork delivers the seed
1. `node scripts/import-acls-seed.js <new-seed.json> --dry-run` → review counts
   ("Scenarios: … (N with narrative content)").
2. Run it for real → narrative lands on the cert-tagged records.
3. Grader Phase 2 + scenario library then show the real case content.
